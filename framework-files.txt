# framework-files.txt
#
# The machine-readable list of UPSTREAM-OWNED framework files, used by
# sync-framework.sh to decide what to diff against the latest semantic-autoencoder.
#
# Rules:
#   - one path per line, relative to the repository root
#   - blank lines and lines starting with '#' are ignored
#   - to add/remove a framework file, edit this list (it syncs like any other framework file)
#
# In the markered subset (agents, skills, rules, CLAUDE.md docs), each file also carries:
#   <!-- FRAMEWORK FILE: improvements → PR to semantic-autoencoder -->
# That header is the human-facing reminder; this manifest is the source of truth the
# script reads.
#
# Deliberately NOT listed (project-owned — never overwritten by a sync):
#   - README.md                              each project writes its own
#   - .claude/settings.json                  per-project permissions
#   - config/project_config.yaml             generated by /setup (gitignored)
#   - .claude/skills/{explain-domain,run-legacy,run-decoded}/   domain skills
#   - encoded/ semantic_ir/ decoded/ regression_tests/ artifacts/ workspaces/ logs/
